## 2024-06-14 - Keyboard accessible table headers without overriding roles
**Learning:** When modifying interactive custom components like `.cyber-table` sortable headers, it's a common accessibility mistake to add `role="button"` directly to the `<th>` elements. Doing so will override the native `columnheader` role and invalidate the `aria-sort` attribute, making it harder for screen readers to properly interpret the element's purpose and state.
**Action:** Always maintain native elements' roles. Implement interaction for non-button interactive elements by manually adding `tabIndex={0}`, `onKeyDown` handlers (specifically to catch 'Enter' and 'Space' keys), and focusing on visually distinct `:focus-visible` styling, all while ensuring proper ARIA attributes like `aria-sort` are included.
